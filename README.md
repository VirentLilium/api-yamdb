## О проекте
Проект представляет собой API для платформы YaMDb, которая собирает отзывы пользователей на произведения.

Проект построен на Django REST Framework и ориентирован на практику построения API с авторизацией и правами доступа.

Произведения делятся на категории.

Произведению может быть присвоен жанр из списка предустановленных.

Добавлять произведения, категории и жанры может только администратор.

Пользователи оставляют к произведениям отзывы и ставят оценку в диапазоне от одного до десяти (целое число). Из пользовательских оценок формируется усреднённая оценка произведения — рейтинг. 

На одно произведение пользователь может оставить только один отзыв.

Пользователи могут оставлять комментарии к отзывам других пользователей.

## Структура проекта

api/ — REST API (viewsets, urls, serializers, permissions, validators, filters)

reviews/ — models (жанры, категории, произведения, отзывы, комментарии), admin

users/ — переопределена модель User, определены роли (admin, moderator, user)

## Аутентификация

Используется SimpleJWT

### Регистрация
POST /api/v1/auth/signup/

### Получение токена
POST /api/v1/auth/token/

## Права доступа

- Чтение доступно всем пользователям (включая неаутентифицированных)
- Авторизованные пользователи могут создавать отзывы и комментарии
- Административные сущности (жанры, категории, произведения) создает только администратор
- Редактировать и удалять контент могут только его автор, администратор и модератор

## Основной функционал API

### Категории (Categories)
- Просмотр списка категорий
- Создание и удаление категорий (только администратор)
- Уникальный slug для каждой категории

### Жанры (Genres)
- Просмотр списка жанров
- Создание и удаление жанров (только администратор)
- Уникальный slug для каждого жанра

### Произведения (Titles)
- Создание, просмотр, редактирование и удаление произведений
- Привязка произведения к категории и жанрам
- Поддержка множественных жанров (many-to-many)
- Автоматический расчёт рейтинга на основе отзывов
- Фильтрация по категории, жанру, названию и году выпуска

### Отзывы (Reviews)
- CRUD для отзывов к произведениям
- Один пользователь может оставить только один отзыв на одно произведение
- Автоматическое назначение автора и произведения
- Ограничение оценки в диапазоне от 1 до 10
- Расчёт рейтинга произведения на основе отзывов

### Комментарии (Comments)
- CRUD для комментариев к отзывам
- Комментарии привязаны к конкретному отзыву и произведению
- Автоматическое назначение автора, отзыва и даты публикации
- Возможность обсуждения каждого отзыва отдельно

### Пользователи
- Регистрация и аутентификация через JWT
- Разделение ролей: пользователь, модератор, администратор
- Автоматическое назначение роли при создании пользователя
- Управление доступом через кастомные permissions

## Дополнительно

## Установка и запуск

### Предварительные требования

- Python 3.10+
- Django 5.1.1
- Django REST Framework 3.15.2

### Шаги установки

1. **Клонирование репозитория**
```bash
  git clone https://github.com/VirentLilium/api-yamdb.git
```
```bash
  cd api-yamdb
```

2. **Создание витруального окружения**
```bash
  python -m venv venv
```

3. **Активация виртуального окружения**

Windows:
```bash
  venv\Scripts\activate
```
Linux:
```bash
  source venv/bin/activate
```

4. **Обновление пакетного менеджера pip и установка зависимостей**
```bash
  python -m pip install --upgrade pip
```
```bash
  pip install -r requirements.txt
```

5. **Применение миграций**
```bash
  python manage.py migrate
```

6. **Запуск сервера разработки**
```bash
  python manage.py runserver
```
7. **Дополнительно: наполнение базы данных**

В директории /api_yamdb/static/data содержатся файлы в формате csv с контентом для ресурсов Users, Titles, Categories, Genres, Reviews и Comments.

Для комфортного тестирования API вы можете использовать management-команду load_csv, добавляющую данные в БД через Django ORM.

Из директории /api_yamdb выполните команду:
```bash
python manage.py load_csv
```

## Документация
Обмен данными с API происходит в формате JSON.

Чтобы ознакомиться с документацией и эндпоинтами, перейдите по адресу http://127.0.0.1:8000/redoc/, предварительно запустив сервер разработки.

### Примеры взаимодействия с API

1. Запрос для получения списка произведений:
GET /api/v1/titles/

Пример ответа от API:
```
{
    "count": 1,
    "next": null,
    "previous": null,
    "results": [
        {
            "id": 1,
            "name": "Интерстеллар",
            "year": 2014,
            "rating": 9,
            "description": "Научно-фантастический фильм",
            "genre": [
                {
                    "name": "Фантастика",
                    "slug": "sci-fi"
                },
                {
                    "name": "Драма",
                    "slug": "drama"
                }
            ],
            "category": {
                "name": "Фильм",
                "slug": "movie"
            }
        }
    ]
}
```
2. Запрос для редактирования отзыва к произведению: 
PATCH /api/v1/titles/5/reviews/7/

Authorization: Bearer <access_token>
Content-Type: application/json

Передаваемые параметры:

- **title_id**: integer - id произведения в БД

- **review_id**: integer - id отзыва к произведению в БД

Пример тела PATCH-запроса к API:
```
{
    "text": "Пересмотрел Интерстеллар спустя несколько лет. Прекрасное кино!",
    "score": 10
}
```

Пример ответа от API:
```
{
    "id": 7,
    "text": "Пересмотрел Интерстеллар спустя несколько лет. Прекрасное кино!",
    "author": "user1",
    "score": 10,
    "pub_date": "2026-06-05T17:42:18Z"
}
```
3. Запрос для получения комментария к отзыву:
GET /api/v1/titles/5/reviews/7/comments/11/

Передаваемые параметры:

- **title_id**: integer - id произведения в БД

- **review_id**: integer - id отзыва к произведению в БД

- **comment_id**: integer - id комментария к отзыву в БД

Пример ответа от API:
```
{
    "id": 11,
    "text": "Отличная рецензия, достойный разбор фильма.",
    "author": "user97",
    "pub_date": "2019-08-24T14:15:22Z"
}
```
